## Project Overview

Sinuzoid is a self-hosted music management and streaming platform built as a microservices architecture. It consists of three backend services, a React frontend, and an Nginx reverse proxy, all orchestrated via Docker Compose.

## Quick Start

```bash
cp .env.example .env
docker compose build && ./start.sh
```

Service URLs after startup:
- Frontend: http://localhost:3000
- Nginx gateway: http://localhost:8080
- FastAPI (music API): http://localhost:8000 — interactive docs at `/docs`
- Symfony (auth): http://localhost:9000
- PgAdmin: http://localhost:8081

## Architecture

```
Frontend (React) → Nginx (8080) → FastAPI (8000) — music/files/playlists
                                → Symfony (9000) — authentication/JWT
                                         ↓
                               PostgreSQL (5432)
```

**Request flow:**
1. Frontend authenticates via Symfony (`/auth/*` routes), receives a JWT stored in sessionStorage (access token) and localStorage (refresh token).
2. All FastAPI requests include `Authorization: Bearer <token>`.
3. FastAPI validates the JWT by proxying `GET http://auth:80/api/me` to Symfony — **FastAPI never validates JWT signatures itself**.

**Nginx routing** (`nginx.conf`): `/` → frontend, `/api` → FastAPI, `/auth` → Symfony.

**Docker network**: all services share `app-network`. Internal service names: `db`, `auth`, `api`, `frontend`.

## Development Commands

### Docker (recommended)

```bash
docker compose up -d              # Start all services
docker compose up <service>       # Start specific service (frontend/api/auth/db)
docker compose logs -f <service>  # Tail logs
docker compose down               # Stop all
./scripts/show-ports.sh           # Print service URLs
```

Reset the database:
```bash
docker compose down
docker volume rm sinuzoid_postgres_data
docker compose up -d
```

### Frontend (`frontend/`)

```bash
bun install
bun run dev       # Vite dev server on port 5173
bun run build     # TypeScript check + production build
bun run lint      # ESLint
```

Frontend environment variables: `VITE_API_URL`, `VITE_AUTH_URL` (see `frontend/.env.example`).

> Several hardcoded `http://localhost:8000` / `http://localhost:9000` URLs still exist in service files and hooks, not yet migrated to env vars.

### FastAPI (`backend/fastapi-api/`)

```bash
uv pip install -r requirements.txt
uvicorn main:app --reload --host 0.0.0.0 --port 8000
# Interactive docs: http://localhost:8000/docs
```

Set `SQL_DEBUG=true` in env to log all SQLAlchemy queries.

### Symfony Auth (`backend/symfony-auth/`)

```bash
composer install
php bin/console lexik:jwt:generate-keypair   # Required on first local run
php bin/console doctrine:migrations:migrate  # Apply migrations
php bin/console make:migration               # Generate migration from entity changes
php bin/console doctrine:migrations:status   # Check migration state
symfony server:start --port=9000
# API docs (API Platform): http://localhost:9000/api/docs
```

The Dockerfile handles key generation and migrations automatically — manual steps only needed for non-Docker development.

## Key Architectural Patterns

### Authentication Flow
FastAPI validates tokens by calling `GET http://auth:80/api/me` via httpx (10s timeout). It never validates JWT signatures locally. All service layer methods re-verify ownership against the DB — the token is never trusted alone.

**Token storage**: access tokens in sessionStorage (1hr, cleared on tab close), refresh tokens in localStorage (30 days). Frontend auto-refreshes within 5 minutes of expiry, polling every 60s.

### Data Model Conventions
- **Track IDs**: UUIDs (generated by FastAPI)
- **User IDs**: integers (generated by Symfony, used as FK in FastAPI models)
- **Metadata**: single JSONB column in the `Metadata` table — avoids schema migrations when new tags are added. Supports 50+ fields extracted via Mutagen.
- **File paths**: `/storage/audio/{user_id}_{uuid}.{ext}` — filename extracted as `.split('/').pop()`

### Audio & Image Fetching
All API resources (audio files, cover images) require a Bearer token. You **cannot** use plain `<img src="...">` — images are fetched with auth headers and converted to blob URLs via `URL.createObjectURL()`.

Audio streaming supports HTTP Range requests → 206 Partial Content for seekable playback.

### Cover Storage Layout
```
/storage/cover/
├── {basename}_cover.jpg          # Original JPEG
├── {basename}_thumb_small.webp   # 150×150
├── {basename}_thumb_medium.webp  # 300×300
└── {basename}_thumb_large.webp   # 600×600
```

### FastAPI Route Pattern
Handlers are static methods returning async closures, grouped into route classes:
```python
class PlaylistCrudRoutes:
    @staticmethod
    def create_playlist(...):
        async def endpoint(data, db, current_user):
            ...
        return endpoint
```

### Frontend State Management
Three Zustand stores:
- **musicStore** — tracks, albums, image caches. 5-min data TTL, 30-min image TTL. Persisted to sessionStorage, **excludes blob URLs**.
- **audioPlayerStore** — current track, queue, volume, shuffle, repeat (none/track/playlist).
- **playlistStore** — user playlists CRUD, 5-min cache TTL.

Background sync runs every 5 minutes and on tab focus / network reconnect.

### No Tests
There is no testing framework (no pytest, jest, or test files). No CI/CD pipelines.

## Service Details

### FastAPI (`backend/fastapi-api/`)

- `main.py` — app entry point, CORS config, router registration
- `app/routes/` — endpoints: `files`, `playlists`, `statistics`, `search`
- `app/services/` — business logic: auth validation, storage, statistics
- `app/models/models.py` — all SQLAlchemy ORM models (single file)
- `app/schemas/schemas.py` — all Pydantic schemas (single file)
- `app/dependencies/auth.py` — JWT validation via Symfony proxy
- `app/database.py` — SQLAlchemy engine/session setup

### Symfony Auth (`backend/symfony-auth/`)

- `src/Controller/AuthController.php` — all endpoints (login, register, `/api/me`, refresh, logout, password reset)
- `src/Entity/` — `User` (int PK, shared with FastAPI), `RefreshToken`, `PasswordResetToken`
- `src/Service/` — `UserService`, `AuthenticationService`, `PasswordService`, `RefreshTokenService`
- `config/` — JWT config (LexikJWT), CORS (NelmioCors), API Platform

JWT config: user identifier is **email**; token extraction via Authorization header only. Refresh tokens are random 64-char hex strings (not JWT), max 5 per user.

### Frontend (`frontend/src/`)

- `pages/` — route-level components
- `components/` — organized by feature: `player/`, `playlists/`, `library/`, `tracks/`, `ui/`, `auth/`
- `store/` — Zustand stores (musicStore, audioPlayerStore, playlistStore)
- `services/` — API call functions (trackApi, playlistApi, metadataApi, apiService)
- `contexts/` — AuthContext, AudioContext (exposes `seekTo()`, `isReady()`)
- `hooks/` — useAudioElement (owns `<audio>` DOM), useAudioPlayer, useMediaSession, useBackgroundSync, etc.
- `types/` — TypeScript type definitions

Key libraries: React 19, React Router 7, Tailwind CSS 4, Zustand 5, JSZip (playlist/library downloads as zip).

Duration format: parsed from ISO 8601 (`PT3M45S`) and `MM:SS`.

## Environment Configuration

Copy `.env.example` to `.env` at the repo root. Key variables:

| Variable | Purpose |
|---|---|
| `POSTGRES_PASSWORD` / `POSTGRES_USER` / `POSTGRES_DB` | Database credentials |
| `DATABASE_URL` | SQLAlchemy connection string |
| `VITE_API_URL` / `VITE_AUTH_URL` | Frontend API endpoints |
| `AUTH_SERVICE_URL` | FastAPI → Symfony internal URL (`http://auth:80`) |
| `APP_SECRET` | Symfony app secret |
| `JWT_PASSPHRASE` | Passphrase for JWT key pair |
| `STORAGE_PATH` | Path for audio/cover storage (`/storage`) |

The Symfony Dockerfile auto-generates JWT keys and runs migrations on startup.

## Tooling

Use **`bun`** / **`bunx`** instead of `npm` / `npx`, and **`uv`** instead of `pip` for all dependency management.

## Database

PostgreSQL 15. Schema initialized from `sinuzoid_database.sql`. Symfony and FastAPI share the same PostgreSQL instance.

Connect from host: `localhost:5432`, database `sinuzoid_db`, user `postgres`.
Connect from containers: host `db`, same port/credentials.

```bash
docker compose exec db psql -U postgres -d sinuzoid_db
```
