## Commands

```bash
composer install
php bin/console lexik:jwt:generate-keypair        # Generate JWT keys (required on first run)
php bin/console doctrine:migrations:migrate        # Apply DB migrations
php bin/console make:migration                     # Generate migration from entity changes
php bin/console doctrine:migrations:status         # Check migration state
symfony server:start --port=9000                   # Dev server
```

The Dockerfile auto-generates JWT keys and runs migrations on startup, so manual steps are only needed for local (non-Docker) development.

Environment variables (see `.env.example`):

| Variable | Purpose |
|---|---|
| `APP_SECRET` | Symfony app secret |
| `DATABASE_URL` | PostgreSQL connection |
| `JWT_SECRET_KEY` | Path to private PEM key |
| `JWT_PUBLIC_KEY` | Path to public PEM key |
| `JWT_PASSPHRASE` | Passphrase for the key pair |

## Architecture

### Endpoints

All endpoints live in `src/Controller/AuthController.php`. The service has no admin-specific controller â€” admin access is enforced via Symfony security attributes.

| Method | Path | Auth required | Purpose |
|---|---|---|---|
| POST | `/api/login` | No | Email + password â†’ JWT + refresh token |
| POST | `/api/register` | No | Create user |
| GET | `/api/me` | Yes | Return current user info |
| POST | `/api/refresh-token` | No | Exchange refresh token â†’ new token pair |
| POST | `/api/logout` | Optional | Revoke refresh token(s) |
| POST | `/api/change-password` | Yes | Change own password |
| POST | `/api/request-password-reset` | No | Request reset token (always returns success) |
| POST | `/api/reset-password` | No | Apply reset using token |

API docs (auto-generated by API Platform): `/api/docs`.

### JWT Configuration

Configured in `config/packages/lexik_jwt_authentication.yaml`. Key details:

- User identifier claim: **email** (not `sub` or `id`).
- Token extraction: **Authorization header only** (query params and cookies are disabled).
- Token lifespan: **3600 seconds** (hardcoded in `AuthController`, not in YAML).
- Keys live at `config/jwt/private.pem` and `config/jwt/public.pem` (generated by `lexik:jwt:generate-keypair`).

FastAPI validates tokens by calling `GET /api/me` on this service â€” it does not validate JWT signatures itself.

### Token Strategy

**Access tokens** (JWT, 1 hour) are generated by LexikJWT and signed with the private key.

**Refresh tokens** are random 64-char hex strings (not JWTs) stored in the `refresh_tokens` table:

- Expire after **30 days**.
- **Max 5 active tokens per user**; oldest are auto-revoked when the limit is reached.
- Marked as `used_at` when exchanged; old token is immediately revoked.

**Password reset tokens** are similar random tokens stored in `password_reset_tokens`:

- Expire after **1 hour**, one-time use.
- All existing reset tokens for a user are invalidated after a successful reset.
- The `request-password-reset` endpoint always returns success (prevents email enumeration).

### Roles & Access Control

```php
// User::getRoles() always includes ROLE_USER
public function getRoles(): array {
    $roles = ['ROLE_' . strtoupper($this->role)]; // 'user' â†’ ROLE_USER, 'admin' â†’ ROLE_ADMIN
    $roles[] = 'ROLE_USER';
    return array_unique($roles);
}
```

Access control rules in `config/packages/security.yaml`:
- `/api/login`, `/api/register`, `/api/request-password-reset`, `/api/reset-password`, `/api/refresh-token`, `/api/docs` â†’ `PUBLIC_ACCESS`
- All other `/api/*` â†’ `IS_AUTHENTICATED_FULLY`

### Error Handling

`src/EventListener/ExceptionListener.php` catches all exceptions and formats them as:

```json
{ "error": { "code": "TOKEN_EXPIRED", "message": "...", "status": 401 } }
```

In `APP_ENV=dev`, a `debug` key is added with exception class, file, line, and trace. JWT exceptions (`ExpiredTokenException`, `InvalidTokenException`, `JWTDecodeFailureException`) are mapped to specific error codes.

### Password Lifecycle

Passwords are auto-hashed on `prePersist`/`preUpdate` via `src/EventListener/UserPasswordHashListener.php` â€” never hash passwords manually in controller code.

`src/Service/PasswordService.php` enforces: minimum 8 characters, at least one uppercase, one lowercase, one digit.

### Database Schema

Three tables managed by this service:

- **`users`**: `id` (serial), `username` (unique), `email` (unique), `password_hash`, `role` (`'user'` | `'admin'`), `created_at`, `updated_at`, `last_login`.
- **`refresh_tokens`**: `id`, `user_id` (FK), `token` (unique), `expires_at`, `is_revoked`, `used_at`.
- **`password_reset_tokens`**: `id`, `user_id` (FK), `token` (unique), `expires_at`, `is_used`, `used_at`.

The `users` table is also read by FastAPI (shared PostgreSQL instance). The `user_id` integer from this table becomes the foreign key used across all FastAPI models.
